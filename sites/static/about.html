About Our Webserver

Welcome to our custom HTTP/1.1 webserver implementation! This project was built from scratch as part of the 42 curriculum, demonstrating a deep understanding of network programming, HTTP protocol, and modern C++ development.

Project Features

Our webserver is a fully functional HTTP/1.1 compliant server with support for:

  • Non-blocking I/O using epoll
  • Keep-alive connections for improved performance
  • Virtual host support (multiple domains on one server)
  • CGI support for dynamic content (Python, PHP, Bash)
  • File uploads with configurable size limits
  • Custom error pages
  • Directory listing (autoindex)
  • Request timeout handling

Technical Architecture

The server is built with a clean, modular architecture separating concerns across multiple layers:

  • Webserver Layer: Handles epoll event loop, connection management, and timeouts
  • Server Layer: Manages virtual hosts, request parsing, and response generation
  • HTTP Parser: Incremental state machine for parsing partial HTTP requests
  • Response Handler: Routes requests and generates appropriate responses
